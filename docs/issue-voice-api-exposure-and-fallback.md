# Issue：前端需暴露语音相关 API 或提供 Fallback，便于在阿里百炼/灵积开通对应服务

## 问题简述

「配音音色（无音色）→ AI 智能设计」使用的是阿里云 **灵积 DashScope** 的**语音定制**能力，具体模型为 `qwen3-tts-vd-realtime-2025-12-16`（接口层面为 `qwen-voice-design`）。当前前端与配置页未告知用户需要在阿里百炼/灵积控制台开通**哪一项服务或模型**，用户仅看到「配置 Qwen API Key」，容易在控制台不知道该开通哪个产品，导致 Key 有权限但仍报错或不可用。

## 现状与依据

- **实现位置**：`src/lib/qwen-voice-design.ts`  
  - 请求地址：`https://dashscope.aliyuncs.com/api/v1/services/audio/tts/customization`  
  - 使用模型：`qwen-voice-design`，目标模型 `qwen3-tts-vd-realtime-2025-12-16`
- **配置入口**：个人设置 → API 配置 → Qwen，当前教程文案为「进入阿里云百炼平台，获取 API Key」（`qwen_step1`），未说明语音设计依赖的**具体 API/模型**。
- **用户心智**：用户只知道「用了 Qwen」，不知道需要在控制台开通「语音定制 / qwen-voice-design / qwen3-tts-vd-realtime-2025-12-16」中的哪一类，容易误以为只开「对话」或「通用 TTS」即可。

## 期望行为（二选一或同时做）

### 方案 A：前端暴露当前使用的语音 API / 模型

- 在**使用到该能力的入口**（如配音音色 → AI 智能设计）或 **Qwen 配置卡片/教程**中，明确写出：
  - 功能与模型的对应关系，例如：  
    **「配音音色 AI 智能设计」使用模型：`qwen3-tts-vd-realtime-2025-12-16`（语音定制 / qwen-voice-design）**
  - 可选：提供「在阿里百炼/灵积控制台开通该模型」的直达链接或说明（若官方有固定链接）。
- 这样用户在阿里百炼/灵积控制台可以按模型名或服务名精确开通，减少试错。

### 方案 B：Fallback 机制

- 当 `qwen3-tts-vd-realtime-2025-12-16` 或当前语音定制接口不可用（如未开通、限流、下线）时：
  - 在**不制造假数据、不静默跳过**的前提下（符合 AGENTS.md），可考虑：
    - 返回明确错误码与文案，提示用户「请在阿里百炼/灵积控制台开通语音定制服务及模型 `qwen3-tts-vd-realtime-2025-12-16`」；
    - 或若产品侧支持多模型/多端点，在配置或运行时选择可用的 fallback 模型并明确记录在日志/计费中，且对用户可见（例如设置项中列出「当前使用：xxx 模型」）。
- 不要求必须做多模型 fallback，但若仅做「错误提示优化」，也应在提示中带上模型名或服务名，便于用户在控制台精准开通。

## 建议实现要点

1. **文案与入口**  
   - 在 Qwen 配置相关 UI（如 `apiConfig` 的 `qwen_step1` 或教程 steps）中补充说明：  
     「配音音色 AI 智能设计」依赖 **语音定制** 能力，模型：`qwen3-tts-vd-realtime-2025-12-16`。  
   - 若存在灵积/百炼「语音定制」或该模型的独立开通页，可在前端提供链接。

2. **错误提示**  
   - 当语音设计接口返回未开通/权限/模型不可用时，错误信息应包含模型名或服务名（如 `qwen3-tts-vd-realtime-2025-12-16` 或「语音定制」），避免用户只看到「请求失败」而不知道开哪个。

3. **可选 Fallback**  
   - 若产品允许「当主模型不可用时改用其他模型」，需在设置或运行逻辑中明确当前使用的模型，并保证计费与日志一致；不做静默降级、不隐藏真实错误原因。

## 验收预期

- 用户在使用「配音音色 → AI 智能设计」或配置 Qwen 时，能在界面或错误提示中看到**具体使用的模型名**：`qwen3-tts-vd-realtime-2025-12-16`（或等价的服务/能力名称）。
- 用户在阿里百炼/灵积控制台能根据前端给出的模型/服务名，准确找到并开通对应能力，减少「已配 Key 仍不可用」的困惑。
